# PromptW — Раздел шаблонов «Авто» (cars) — дизайн

> Статус: утверждён пользователем 2026-06-24. Следующий шаг — план реализации
> (writing-plans). Канон по шаблонам — `docs/template-authoring-and-face-fidelity.md`.

## 1. Цель

Новый раздел из **10 креативных авто-шаблонов** с вау-эффектом: человек + крутая
машина в эффектных локациях (флагман — бирюзовый Гелик в пустыне Дубая). Все шаблоны
кастомизируемы и подходят **и М, и Ж** через пол-пилюлю и gender-keyed образы. Один
шаблон (советский) позволяет выбирать **модель машины** и **локацию**. Модель —
NanoBanana PRO. Сохранение лица — строго по гайду (identity-блок, кадр поясной/средний,
без праймингов).

## 2. Категория

- Новая категория `cars`, лейбл **«Авто» / «Cars» / «Coches»**.
- `webapp/static/js/app.js`: добавить `"cars"` в `TPL_CAT_ORDER` после `"men"` →
  `["girls","men","cars","animals","kids","travel","cinema","jobs","eras"]`.
- `webapp/static/js/i18n.js`: добавить ключ `cars` в ru/en/es (рядом с `girls/men/…`),
  паритет ключей обязателен.
- Бамп `?v=N` для `app.js` и `i18n.js` в `index.html` (статика служится live).
- Флагман `car-dubai-gwagon-photo` → `featured:true` (верхняя сетка «Тренды»);
  остальные `featured:false`.

## 4. Схема params (шаблоны 1–9)

Контролы — `sheet` (как у существующих), кроме `gender` (`pills`). Каждая опция —
`{value,label:{ru,en,es},frag}`. `frag` — русский фрагмент промта.

1. **`gender`** (`pills`, первый): опции `male`/`female` с `grammar`:
   - male: `{noun:"мужчина", adjE:"ый"}`
   - female: `{noun:"девушка", adjE:"ая"}`
   - Движок подставит `{subject}` = «<noun> тот же самый человек, что на загруженном
     фото-референсе, с его реальными чертами лица». Age-param НЕ добавляем (чистый noun).
2. **`color`** (`sheet`): цвет фирменной машины, 4 опции (для Гелика —
   Бирюзовый / Чёрный мат / Белоснежный / Песочный; для других — подобрать 4 уместных).
   `frag` описывает цвет+лак, напр. «бирюзового цвета с матовым премиальным покрытием».
3. **`location`** (`sheet`): 2–4 локации из таблицы §3. `frag` — описание фона.
4. **`outfit`** (`sheet`, **gender-keyed**: `dependsOn:"gender"`,
   `options:{male:[…],female:[…]}`): по 4 образа на пол, подобранных под вайб сцены.
   Базовый набор М: кожаная куртка / премиум-худи оверсайз / костюм по фигуре /
   пальто+водолазка. Базовый Ж: атласный слип-дресс / кожаный тренч / бомбер+джинсы /
   вечернее платье. Адаптировать под сцену (напр. для off-road — пуховик/парка).
5. **`glasses`** (`sheet`): Авиаторы / Вайфареры / Без очков / Очки на лбу.

`buildTplPrompt` подставляет фрагменты в `{gender→subject}/{color}/{location}/{outfit}/
{glasses}`. «Без значения»-опции (напр. «Без очков») имеют реальный `frag`
(«без очков, лицо полностью открыто»), не пустой — иначе в промте «. .».

## 5. Шаблон 10 — советский (особый)

Params: `gender` + **`car`** (вместо `color`) + `location` + `outfit` + `glasses`.

- **`car`** (`sheet`): Волга ГАЗ-21 / Волга ГАЗ-24 / Москвич-412 / Жигули ВАЗ-2101
  «копейка» / Чайка ГАЗ-13 / Нива ВАЗ-2121 / УАЗ-469 / ЗИЛ-114. `frag` — модель +
  характерный заводской цвет/детали эпохи.
- **`location`** (`sheet`): советский двор у хрущёвок / заснеженная Москва (Красная
  площадь вдали) / южный курорт СССР (Сочи-Крым, пальмы) / просёлок в летнем поле /
  ночной ретро-город под фонарями.
- **`outfit`** (gender-keyed): ретро-одежда 60–80-х (М: кожаная/замшевая куртка,
  водолазка, костюм-тройка той эпохи; Ж: ретро-платье в горошек, пальто с поясом,
  косынка-образ, твидовый костюм).
- **Вайб — тёплая ностальгия, не ирония:** плёночная эстетика (зерно, мягкий тёплый
  свет, лёгкая выцветшесть), без сатиры.

## 6. Промт каждого шаблона (face-fidelity)

`prompt` == `skeleton` (как у всех параметрических: оба содержат слоты). Структура:

1. **Identity-блок во главе** — канон из боевых шаблонов (см.
   `cine-superhero-photo`/`man-supercar-photo`): «КЛЮЧЕВОЕ ТРЕБОВАНИЕ: точно перенести
   лицо… скопируй лицо 1:1: форма и ШИРИНА лица/щёки (не сужать), челюсть, нос, разрез
   и цвет глаз, брови, губы, скулы, кожа, родинки, щетина/борода и причёска…
   это обычный человек, а НЕ модель/знаменитость; без AI-сглаживания/омоложения…».
   Через `{subject}` адаптируется по полу.
2. **Сцена**: `{subject}` у/на машине; машина = <фирменная> {color} (или {car} для №10);
   `{outfit}`; `{glasses}`; локация = {location}. **Поза запинена** под концепт, напр.
   «сидит НА капоте, опираясь руками сзади, ноги на бампере» / «сидит на крыше, нога
   свесена» — чтобы модель не вписывала человека сквозь кузов/в воздух.
3. **Композиция**: «поясной/средний план, человек крупно, ЛИЦО в центре и в фокусе;
   НЕ мелкая фигура в полный рост с маленьким лицом».
4. **Технично-нейтрально**: объёмный свет, боке, отражения на лаке; фотореализм, кожа
   с порами. **Без праймингов** («успешный/статусный/супермодель/VOGUE/эффектный/
   красивый»). 9:16. «Без посторонних людей, без текста, букв, логотипов и водяных
   знаков.»
5. Авто-слой `_augment_template_prompt` (де-прайминг + EN identity-якорь/суффикс +
   перекадровка) применяется поверх (т.к. есть `settings.tplId` + `photo-refs`) —
   страховка, не замена. Промты пишем чисто сразу.

`desc{ru,en,es}` — короткая подсказка («Загрузите фото с чётко видимым лицом — оно
сохранится с референса. Выберите пол, цвет машины, локацию, образ и очки»).

## 7. Превью (самогенерация)

- Пайплайн — `promptw-preview-pipeline`: референсы **`media/man.jpg`** (М) и
  **`media/girl.jpeg`** (Ж) [локально; на VPS соответствуют refman.jpg/refmodel.jpg].
  `_shrink_image ≤1536px` покрывает оверсайз girl.jpeg.
- Для каждого из 10 — сгенерировать превью по дефолтным параметрам шаблона через прод
  NanoBanana PRO, скачать результат в `webapp/static/tpl/<id>.jpg`, **закоммитить в git**
  (не писать в git-чекаут VPS в рантайме). `preview.img`/`preview.full` →
  `/static/tpl/<id>.jpg`.
- Гендер превью: для «мужских по вайбу» сцен брать man.jpg, для «женских» — girl.jpeg;
  флагман-Гелик — на усмотрение (предложить оба, выбрать лучший).
- До готовности превью шаблоны `enabled:false`; после заливки превью → `enabled:true`.

## 8. Доставка и деплой

- **Данные** → 10 новых объектов в `db/templates_seed.json` (сид `_seed_templates`
  `ON CONFLICT (id) DO NOTHING` — деплой только добавит). Цена авторитетна на сервере
  из `templates.cost`.
- **Клиент**: `TPL_CAT_ORDER` + лейблы `cars` (i18n) + бамп `?v=N`.
- **Метод**: ветка → PR → `gh pr merge` → `bash srv2.sh gitdeploy` (когда DNS на VPS
  стабилен — сейчас на временном фиксе resolv.conf; координировать). Бэкенд-данные
  требуют рестарта сервиса (сид при старте); статика — нет.
- Не деплоить без нужды во время рендера видео.

## 9. Чеклист готовности (на каждый шаблон, из гайда §10)

- [ ] Промт описывает только сцену/машину/локацию/образ; внешность — отсылкой к
      референсу, без хардкода черт.
- [ ] Identity-блок front-load; `{subject}` от gender-grammar.
- [ ] Нет позитивных праймингов.
- [ ] Кадр поясной/средний, лицо крупно; нет «в полный рост».
- [ ] Поза запинена (человек на/у машины, не сквозь кузов).
- [ ] `refField`/поле — фото-шаблон грузит `photo-refs` (дефолт для photo).
- [ ] `params` собраны; gender-keyed `outfit` имеет male/female; «без значения»-опции
      с реальным `frag`.
- [ ] Превью сгенерировано/закоммичено; `category:"cars"`, `cost:30`, `sort_order`.
- [ ] Протестировано 2–3 генерациями на КАЖДЫЙ пол (стохастика!), оценивалась подмена
      личности, а не качество сцены.

## 10. Вне рамок (YAGNI)

- Видео-авто-шаблоны (только photo).
- Выбор модели машины в шаблонах 1–9 (только цвет; модель фикс — иначе теряется
  фирменность). Выбор модели — только в №10.
- Новые контролы/движок params (используем существующие `pills`/`sheet`/gender-keyed).
- Изменение цены/логики биллинга.
