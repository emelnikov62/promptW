# PromptW — Категории «Дети» и «Путешествия мечты» (дизайн)

> Дата: 2026-06-23. Добавляем две категории шаблонов и 14 шаблонов (все —
> NanoBanana PRO, фото, 9:16, unisex auto-detect, с `params`). Канон по авторингу
> шаблонов и сохранению лица — `docs/template-authoring-and-face-fidelity.md`.

## Цель
1. Закрыть пустую категорию **«Дети»** (`kids`) — 9 шаблонов.
2. Ввести новую совместную (муж+жен) креативную категорию **«Путешествия»**
   (`travel`, «Путешествия мечты») — 5 шаблонов.
3. У каждого шаблона — кураторские `params` (фишка платформы), превью
   самогенерируется через прод NanoBanana PRO.

## Не-цели
- Видео-шаблоны (только фото).
- Перезапись существующих шаблонов (только новые id).
- Прямой UPDATE по prod-таблице `templates` (строкой владеет админка; новый
  контент идёт через `templates_seed.json` + `ON CONFLICT DO NOTHING`).

## Витрина / категории
- `kids` — лейблы уже в `i18n.js` (ru «Дети» / en «Kids» / es «Niños»).
- `travel` — **новые** лейблы в `i18n.js` ×3: ru «Путешествия» / en «Travel» /
  es «Viajes» (короткий лейбл секции; «мечты» — в описаниях шаблонов).
- `TPL_CAT_ORDER` (`app.js`): `["girls","men","kids","travel","cinema","jobs","eras"]`.

## Общие параметры всех 14 шаблонов
`type:photo`, `model:"NanoBanana PRO"`, `ratio:"9:16"`, `ratios:["9:16","3:4"]`,
`minPhotos:1`, `maxPhotos:8`, `cost:30`, `enabled:true`, `featured:false`,
`category` = `kids`|`travel`, `params:[...]` (1–2 шт.), `skeleton`==`prompt`
с слотами параметров, `desc` ru/en/es, `preview.img/full` = `/static/tpl/<id>.jpg`.

## Промты (сохранение лица)
Каждый промт = канонический front-load identity-блок (как `cine-wizard-photo`) +
сцена/одежда/свет + слоты `params` + композиция (поясной/средний план, лицо
крупно, фотореализм, 9:16, без текста/логотипов). Внешность — только из фото,
без хардкода черт (§4.1 канон-гайда).

**Спецзащита для детей** (identity-суффикс): «это РЕБЁНОК — сохрани детские
пропорции лица и возраст строго как на фото, НЕ делай старше/взрослее, без
макияжа, без AI-сглаживания, без идеализации». Главный риск дрейфа у детей —
«состарить/омоделить». Авто-слой `_augment_template_prompt` применяется сверху
штатно (EN-якорь + суффикс + перекадровка), доп. правок кода не требует.

«Принц/принцесса» и «фея/эльф» — наряд адаптируется по полу с фото текстом
промта («платье/корона для девочки, камзол/доспех для мальчика — по полу с
референса»), без отдельного gender-param (unisex-паттерн как cinema).

## Превью (14 шт.) — пайплайн
Через прод NanoBanana PRO на VPS (`promptw-preview-pipeline`):
1. **Детский референс** (отсутствует): сгенерировать синтетический детский
   портрет — 1 девочка + 1 мальчик — через NanoBanana text-to-image (без ref),
   залить в S3 (`storage.put_file`) → публичный URL.
2. **Travel**: переиспользовать взрослые refmodel/refman (перезалить в S3 —
   MEDIA_DIR очищен при миграции).
3. Батч-генерация: prompt-файлы + `tools_gen_batch.py` в `/tmp` на VPS,
   `bash srv2.sh ssh "... venv/bin/python tools_gen_batch.py ..."`.
4. Результаты → `webapp/static/tpl/<id>.jpg` (закоммитить) — НЕ писать в
   git-чекаут в рантайме.
5. Реальную генерацию (KIE-ключ из прод-`.env`) запускает владелец через
   `srv2.sh`; Claude готовит prompt-файлы, скрипт и сид.

## Деплой
Ветка → PR → `gh pr merge` → `bash srv2.sh gitdeploy`. Затрагиваемые файлы:
- `db/templates_seed.json` (+14 объектов).
- `webapp/static/js/app.js` (`TPL_CAT_ORDER` +`kids`/`travel`) + бамп `?v`.
- `webapp/static/js/i18n.js` (3 лейбла `travel`) + бамп `?v`.
- `webapp/templates/index.html` (бамп `?v` для app.js/i18n.js).
- `webapp/static/tpl/*.jpg` (14 превью — можно отдельным PR после генерации).

## Риски / открытые точки
- Стохастика NanoBanana (нет seed): превью брать из лучшего прогона (§6 канона).
- Детский референс должен давать узнаваемо «детское» лицо — проверить, что
  identity-суффикс «не взросли» работает на тест-генерациях.
- Если превью не успеваем сгенерировать к merge — категории покажутся с
  плейсхолдер-плитками (`renderTplCats` рисует 4 пустые), контент дольётся PR'ом.
