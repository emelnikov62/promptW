## 3. Формат шаблона (контракт)

Каждый шаблон — объект в массиве `db/templates_seed.json`:

```json
{
  "id": "<kebab-id>", "type": "photo", "cost": 30, "sort_order": <int>,
  "category": "<girls|men|kids|travel|cinema|jobs|eras>",
  "enabled": true, "featured": false,
  "title": {"ru":"…","en":"…","es":"…"},
  "preview": {"img":"/static/tpl/<id>.jpg","full":"/static/tpl/<id>.jpg"},
  "definition": {
    "model":"NanoBanana PRO","ratio":"9:16","ratios":["9:16","3:4"],
    "minPhotos":1,"maxPhotos":8,
    "prompt":"<identity-block> <scene с {slots}> <composition-tail>",
    "skeleton":"<идентично prompt — те же {slots}>",
    "params":[ {param}… ],
    "desc": {"ru":"…","en":"…","es":"…"}
  }
}
```

- **`prompt` и `skeleton` идентичны** (оба содержат `{slots}`) — как у всех боевых.
- **`sort_order`**: новым присвоить блок `>=100` (по возрастанию внутри категории),
  чтобы они аппендились после существующих. Не пересекать с текущими значениями.
- **`id`-схемы (строго как у существующих в каждой категории, проверено по seed):**
  - kids → **без** `-photo`: `kid-<x>` (напр. `kid-astronaut`)
  - travel → **без** `-photo`: `travel-<x>` (напр. `travel-maldives`)
  - girls → `girl-<x>-photo` · men → `man-<x>-photo`
  - cinema → `cine-<x>-photo` · jobs → `job-<x>-photo` · eras → `era-<x>-photo`
  ⚠️ Точные id каждого — см. §6.

### 3.1 Параметр (`param`)

```json
{ "id":"outfit", "control":"sheet",
  "label":{"ru":"Образ","en":"Look","es":"Estilo"},
  "options":[
    {"value":"summer","label":{"ru":"…","en":"…","es":"…"},"frag":"<RU-фрагмент промта>"}
  ] }
```

- `control`: `"sheet"` (шторка — по умолчанию) или `"pills"` (≤3 коротких опции).
- **Каждая опция несёт реальный `frag` на русском** — фраза, вплетаемая в slot.
  Опция «без …» = реальный frag («без шляпы», «без аксессуаров»), НЕ `frag:""`.
- 2–3 параметра на шаблон. Слоты `{id}` вплетены в сцену прозой со связками
  («На ней {outfit}. В руках {accessory}.»). **Каждый `{slot}` skeleton'а покрыт
  param'ом** — после resolve (slot→frag) не остаётся `{...}`.
- Дефолт = первая опция (движок `tplFirstVal` падает на первую, если нет `default`).
- Gender-keyed опции (`options:{male,female}` + `dependsOn`) НЕ нужны — категории
  girls/men одно-полые, остальные unisex без gender-param.

## 4. Identity-блоки (константы, копировать дословно)

Внешность — ТОЛЬКО из фото (гайд §4.1). Никаких позитивных праймингов
(успешный/статусный/супермодель/VOGUE/красивый). Три варианта по полу категории:

Лицо на результате должно быть таким, чтобы её мгновенно узнали родные. Скопируй лицо 1:1: сохрани без единого изменения форму и ШИРИНУ лица и полноту щёк (НЕ сужай, не утончай, не вытягивай овал), овал и линию челюсти, форму и пропорции носа, форму и разрез глаз, расстояние между глазами, брови, форму и полноту губ, скулы, структуру и текстуру кожи, родинки, веснушки — 100% узнаваемость. БЕЗ сглаживания кожи, БЕЗ AI-эффекта, не идеализируй, не омолаживай, не делай лицо тоньше, симметричнее, красивее или «модельным», не меняй форму носа и пропорции лица. Если сомневаешься — копируй черты точно с референса. Волосы — цвет, длина и причёска строго как на загруженном фото референса.`

Лицо должно быть таким, чтобы его мгновенно узнали родные. Если сомневаешься — копируй черты точно с референса.`

### 4.5 Composition-tail (константа, в конец каждого prompt/skeleton)
> `КОМПОЗИЦИЯ: поясной или средний план, человек крупно в кадре, ЛИЦО — в центре внимания, в фокусе и максимально чёткое; НЕ мелкая фигура в полный рост с маленьким лицом. Фотореализм, реалистичная кожа с порами и текстурой, без ретуши и AI-сглаживания. Вертикальный кадр 9:16. Без посторонних людей, без текста, букв, логотипов и водяных знаков.`

(Для kids «человек» → «ребёнок».)

### 4.6 Ч/Б оговорка (только `cine-noir`)
В сцену добавить: `Чёрно-белая нуар-стилизация НЕ меняет черты лица — это то же лицо
с референса, просто в монохроме; сохрани реальные пропорции и ширину лица.`

## 5. Сборка prompt/skeleton

`prompt = identity-блок(§4.x по категории) + " " + <сцена с {slots}> + " " + composition-tail(§4.5)`.
`skeleton` идентичен. Сцена — 1–3 предложения: что за место/событие, куда вплетены
слоты, нейтральное настроение/свет (без оценочных «роскошный/эффектный» про человека).

## 7. Пайплайн превью

1. **Рефы:** `_shrink_image` (или PIL) `media/girl.jpeg`+`media/man.jpg` ≤1536 по
   длинной стороне → `storage.put_file(..., remove_local=False)` → публичные S3-URL.
   Детские: 2 синтетических лица через NanoBanana **text-to-image** (без `image_input`,
   `aspect_ratio:"3:4"`): «фронтальный студийный портрет улыбающегося ребёнка ~7 лет
   (девочка / мальчик), нейтральный фон, мягкий свет, фотореализм» → S3-URL. Реф под
   тему: ballerina→девочка, police→мальчик, прочие kids — любой подходящий.
2. **Промт превью** = resolved skeleton: каждый `{slot}` → `frag` ПЕРВОЙ опции param.
   **БЕЗ `_augment_template_prompt`** (его проходят только реальные ген-ы юзеров с
   `tplId`+`photo-refs`). Для unisex выбрать реф (м/ж) под тему шаблона.
3. **Генерация** на VPS батчем (паттерн из `promptw-preview-pipeline`): семафор 4,
   `KieGenerator._create_task("nano-banana-pro",{prompt,output_format:"png",
   aspect_ratio:"9:16",image_input:[ref_url]})` → `_poll_task` → `_parse_result_urls`
   → `_download_file`; если вернулся `http`-URL (S3-режим) — тянуть `urllib` с
   `User-Agent: Mozilla/5.0` (голый urllib к CDN KIE → 403).
4. **Постобработка:** ужать до **714×1280 JPEG q86** → `webapp/static/tpl/<id>.jpg`
   (закоммитить файл; НЕ писать в git-чекаут VPS в рантайме).

## 8. Проверка («всё проверь»)

A. **Статика (до генерации):**
   - `json.load(templates_seed.json)` — валиден; нет дублей `id`; ровно +50.
   - У каждого нового: все обязательные ключи; `params` непусты; у каждой опции
     непустой `frag`; **каждый `{slot}` в skeleton покрыт param'ом** (resolve →
     нет остаточных `{...}`); `prompt == skeleton`; identity-блок соответствует §4.x.
B. **Превью:** каждое изображение визуально проверить на **подмену лица** (не «модель»,
   ширина лица, борода у мужчин, возраст), кадрирование (лицо крупно), артефакты.
   Дрейф → пере-ролл 1–2× (стохастика, seed/strength нет). Собрать **контакт-лист**
   (сетка миниатюр по категориям) → **показать пользователю на одобрение**.
C. **После деплоя:**
   - `GET /api/templates` → +50 новых id в нужных категориях.
   - `GET /api/templates/{id}` на выборке → params/skeleton корректны.
   - Smoke: 2–3 реальные генерации с настоящим рефом → лицо переносится (опц., кредиты).

## 9. Деплой

Фронт НЕ трогаем (`TPL_CAT_ORDER` уже содержит 7 категорий; титулы/desc — в seed;
`?v` ассетов бампать не нужно). Только `db/templates_seed.json` + 50 превью в
`webapp/static/tpl/`.

Ветка → PR → `gh pr merge` → на машине владельца `bash srv2.sh gitdeploy`. Сервер при
старте досидит новые шаблоны (`_seed_templates` `ON CONFLICT (id) DO NOTHING` — только
добавит, существующее/админ-правки не тронет). Деплой и прод-генерации — за явной
авторизацией пользователя (production-гейт).

## 10. Подход к реализации (B)

Параллельные субагенты, по одному на категорию (7 шт), заполняют творческий контент
(сцена + опции + frag RU + title/desc/labels RU-EN-ES) в фикс. identity-skeleton по
этой спеке, отдают строгий JSON (schema). Главный агент валидирует (§8.A), собирает в
seed, генерит превью (§7), проверяет (§8.B), контакт-лист → одобрение → деплой (§9).

## 11. Риски

- ~50+ прод-генераций NanoBanana = реальные KIE-кредиты + время (~20–40 мин + ре-роллы).
- Женские лица дрейфуют сильнее; `cine-noir` (Ч/Б) — самый рискованный (оговорка §4.6).
- Стохастичность (нет seed) → повторяемость только промтом + ре-роллами.
- Координация деплоя: не деплоить во время рендера видео; единый метод (gitdeploy, не rsync).
