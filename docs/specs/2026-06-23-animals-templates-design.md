# PromptW — Раздел шаблонов «Животные» (20 шаблонов) — дизайн-спека

> Дата: 2026-06-23. Канонический гайд — `bot-src/CLAUDE.md`; авторинг шаблонов и
> сохранение лица — `bot-src/docs/template-authoring-and-face-fidelity.md` (далее
> «гайд»). Эта спека описывает добавление новой категории `animals` с 20 фото-
> шаблонами «человек + животное», параметризованными на пол.

---

## 1. Цель и объём

Добавить в галерею новый раздел **«Животные»** (`category: "animals"`) с **20
оригинальными фото-шаблонами**: человек в кадре с животным/питомцем, лицо
сохраняется с фото-референса юзера. Микс «вау-экзотики» и домашних питомцев.

- Все 20 — NanoBanana PRO, `type: "photo"`, 9:16 (+3:4), `cost: 30`,
  `minPhotos: 1`, `maxPhotos: 8`, параметризованные (есть `params`/`skeleton`).
- Все работают для **обоих полов** (param `gender`).
- Полный цикл: контент → код (категория + сид) → **20 превью** через прод
  NanoBanana → PR → деплой.

**Не входит в объём:** видео-шаблоны, новые модели, изменения движка params,
правки `_augment_template_prompt`.

## 3. Авторинг-контракт (как устроен каждый шаблон)

Движок параметров — клиентский (`buildTplPrompt`/`initTplParamState` в `app.js`),
проверен по коду. Контракт:

### 3.1 `definition` (JSONB), обязательные ключи
```
model: "NanoBanana PRO"
ratio: "9:16"
ratios: ["9:16","3:4"]
minPhotos: 1
maxPhotos: 8
skeleton: "<identity-блок + сцена со слотами>"
prompt:   "<== идентичен skeleton>"   // fallback; валидатор требует prompt==skeleton
params:   [ ... ]
desc:     {ru,en,es}
```

### 3.2 `skeleton` — правила
- **Identity-блок ВСЕГДА первым и фиксированным** (гайд §4.2). Берём боевой блок
  (как в `birthday-photo`/`man-alpine`), пол-нейтральный:
  > «Это фотография конкретного человека с загруженного референса. Перенеси его
  > лицо без изменений: те же черты лица, форма и ширина лица, нос, глаза, разрез
  > глаз, брови, губы, овал, скулы, оттенок кожи, родинки, щетина/борода и её форма
  > (если есть — сохрани, нет — не добавляй), причёска — 100% портретное сходство,
  > это тот же самый человек, не идеализируй и не омолаживай. ЦВЕТ ГЛАЗ — как на
  > референсе. ВАЖНО: это обычный человек, а НЕ фотомодель — не делай худее /
  > скуластее / моложе / симметричнее, без сглаживания кожи и AI-эффекта.»
- Дальше — **сцена**: ставит человека `{subject}` рядом с конкретным животным,
  **поясной / средний план, лицо крупно** (гайд §4.4 — не «в полный рост»).
- Внешность человека — НЕ описывать словами (гайд §4.1). Никаких праймингов
  (успешный/гламурный/супермодель/VOGUE/красивый — гайд §4.3, §5).
- Поза/расположение животного **пинится явно**, если двусмысленно (гайд §4.4).
- **Слоты:** `{subject}` (спец — собирается движком из `gender.grammar.noun` +
  фикс-хвост про референс; НЕ писать руками), плюс по одному `{<id>}` на каждый
  не-gender param. Любой `{slot}` обязан резолвиться (есть param с таким id).
- Описывать животное как **реальное живое животное в кадре**, естественный мех/
  перья, корректная анатомия; запретить химер/игрушечность.

### 3.3 `params` — рецепт (3–4 контрола на шаблон)
Формат опции: `{value, label:{ru,en,es}, frag}` (`frag` — русский фрагмент промта).
Gender-keyed param: `options:{female:[...],male:[...]}` + `dependsOn:"gender"`.

1. **`gender`** (`control:"pills"`, `default` под превью-реф) — опции `female`/`male`
   с `grammar:{noun,adjE}` (`женщина`/`ая`, `мужчина`/`ый`). Без `frag`.
2. **`outfit`** (`control:"sheet"`, `dependsOn:"gender"`, gender-keyed) — по 3 опции
   на пол, под сцену. Аксессуары (поводок, ошейник-бабочка, плед, букет) вшиваются
   сюда или в позу.
3. **Сценовый param A — `setting`** (`pills`, 3 опции) — вариант локации/фона.
4. **Сценовый param B** — `pills`/`sheet`, 2–3 опции: окрас/вид животного **или**
   поза/расположение (`pose`), под конкретную сцену.

Опции «без значения» дают реальный `frag` (не `""`) — гайд §7.

### 3.4 `desc` {ru,en,es}
Единый шаблон (как у `birthday-photo`): «Загрузите своё фото — лицо сохранится с
референса. Ниже выберите пол, наряд и детали сцены, остальное соберётся
автоматически.» Адаптировать хвост под сцену.

## 5. Привязка категории в коде

1. **`webapp/static/js/app.js`** — `TPL_CAT_ORDER`:
   `["girls","men","animals","kids","travel","cinema","jobs","eras"]` (animals после
   men).
2. **`webapp/static/js/i18n.js`** — лейбл категории в 3 местах (ru/en/es):
   `animals: "Животные"` / `"Animals"` / `"Animales"`.
3. **`db/templates_seed.json`** — 20 объектов добавляются **аддитивно** (raw-append
   перед `]`, без re-dump всего файла → чистый диф). Поля: `id`, `type:"photo"`,
   `cost:30`, `sort_order`, `category:"animals"`, `enabled:true`, `title{ru,en,es}`,
   `preview:{img,full}` → `/static/tpl/animal-<slug>.jpg`, `definition{...}`,
   `featured` (только #1, #12).
4. **`webapp/templates/index.html`** — бамп `?v=` у `app.js` и `i18n.js`.

Сервер досидит новые id при старте (`_seed_templates`, `ON CONFLICT (id) DO
NOTHING`) — существующие шаблоны не трогаются.

## 6. Генерация 20 превью (прод-пайплайн)

Переиспользуем проверенный пайплайн (`promptw-preview-pipeline`, PR #90):

1. **Рефы:** `media/girl.jpeg`, `media/man.jpg` → ужать ≤1536px по длинной стороне →
   `storage.put_file(..., remove_local=False)` → публичные S3-URL.
2. **Батч:** `gen_batch.py` в `/tmp` на VPS (семафор 4), вход `jobs.json` =
   `[{id, ref, prompt}]`, где `prompt` = **resolved skeleton** (каждый `{slot}` →
   первая опция param; `{subject}` → дефолтный пол), **без** `_augment_template_prompt`.
   Запуск `nohup … &`, поллинг `/tmp/prev/_result.json`.
3. **Проверка дрейфа лица:** прод InsightFace (`face_verify.embed()` +
   `similarity()`) результат↔реф по всем 20; cosine `<0.45` = подозрение → пере-ролл
   (женские лица дрейфуют сильнее — гайд §6). Дополнительно глазами: проверять
   подмену личности и реалистичность животного, а не качество сцены.
4. **Постобработка:** ресайз → **714×1280 JPEG q86** →
   `webapp/static/tpl/animal-<slug>.jpg`.
5. **Приватность:** после генерации **удалить рефы с S3** (`storage.delete_url`) —
   это реальные лица на public-read.

**Прод-гейт:** шаги 1–4 жгут KIE-кредиты и крутятся на VPS — запускаются только по
явной команде владельца.

## 7. Деплой и раскатка (через worktree)

Несколько сессий делят рабочий каталог — лендим в отдельном worktree (гайд §9):

1. Авторинг контента 20 шаблонов (возможно через субагентов по подтемам, сборка
   ассемблером + валидация: каждый `{slot}` резолвится, `prompt==skeleton`,
   identity-префикс на месте, нет дублей id).
2. `git worktree add -b feat/animals-templates /tmp/land origin/main`; правки только
   своих файлов: `templates_seed.json`, `app.js`, `i18n.js`, `index.html`.
3. Превью (§6, прод-гейт) → 20 файлов в `webapp/static/tpl/`.
4. Коммит → PR → **по команде владельца** `gh pr merge` + `bash srv2.sh gitdeploy`.
5. Проверка: прод-БД содержит 20 новых id; раздел «Животные» виден в галерее после
   «Мужчины». `git worktree remove`.

## 8. Риски и контроль

- **Дрейф лица в стилизованных/контрастных сценах** (снег, закат, ч/б-подобные) —
  гайд §3. Митигация: identity-блок, без праймингов, поясной план, face-verify
  пере-ролл превью; на проде юзерские ген-ы проходят `_augment` + (если включён)
  Level C face-verify.
- **Нереалистичное/химерное животное** — явный запрет в skeleton, выбор живого
  реального животного; визуальная проверка превью.
- **Конфликты ассетов при параллельных сессиях** — worktree + бамп `?v=` только у
  изменённых файлов.
- **Гейты:** прод-генерация превью и `gitdeploy` — только по явной команде.
